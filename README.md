Ход выполнения работы
В ходе выполнения работы был создан базовый абстрактный класс Animal с общими характеристиками: Кличка, Возраст, Среда обитания, Тип питания. Метод GetInfo() объявлен как virtual для возможности переопределения в классах-наследниках.

На основе базового класса созданы пять классов-наследников, каждый с уникальным свойством:
Mammal (млекопитающее) – добавлено свойство HasFur (наличие шерсти)
Bird (птица) – добавлено свойство WingSpan (размах крыльев)
Fish (рыба) – добавлено свойство WaterType (тип воды)
Reptile (пресмыкающееся) – добавлено свойство IsVenomous (ядовитость)
Amphibian (земноводное) – добавлено свойство SkinMoisture (влажность кожи)

В каждом классе-наследнике метод GetInfo() переопределен с использованием ключевого слова override для добавления информации об уникальных свойствах.

Создан класс ZooManager, реализованный как Singleton. Класс содержит:
Приватный конструктор
Статическое свойство Instance для доступа к единственному экземпляру
Коллекцию List<Animal> для хранения животных
Методы AddAnimal() для добавления животных и DisplayAllAnimals() для вывода информации
Метод ShowMenu() для взаимодействия с пользователем через консольное меню

В методе Main продемонстрирована работа программы: созданы несколько объектов разных подтипов животных, добавлены в менеджер через Singleton, запущено консольное меню с возможностью просмотра информации о животных, добавления новых и выхода из программы.

Выводы
В результате выполнения работы были практически освоены основные принципы ООП: наследование (создание иерархии классов), полиморфизм (переопределение методов), инкапсуляция (скрытие данных). Также успешно применен паттерн Singleton для обеспечения единственного экземпляра менеджера животных. Разработанная программа демонстрирует корректную работу всех механизмов и предоставляет удобный интерфейс для управления коллекцией животных.
